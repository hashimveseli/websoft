Assignment S09

Cross-Site Scripting (XSS) vulnerability lets a hacker execute his own JavaScript code which can lead to the hacker being able to reveal your chats with friends and get your password. This could lead to someones social media being hacked and these days you could gain profit from hacking a famous instagram page or whatever. Security Misconfigurations is the most commonly seen issue and occurs when a system is configured in a way that makes it vulnerable and can happen on any level of the application stack (in the web server, the database, the web application and network equipment and so on). These flaws give attackers unauthorized access to system data or functionality which can lead to very negative outcome. And last but not least, Injection is one of the web security vulnerabilities that can happen in any kind of system such as:

- A PHP page sending data to a MYSQL database

- A Python script using system() commands

- A program sending user input to a mail server

- A car dealership and their contract..

Injection can result in data loss, corruption, or disclosure to unauthorized parties, loss of accountability, or denial of access and sometimes lead to complete host takeover. This makes Injection a dangerous security breach.

My guidelines to follow for a secure web application are:

* You need to protect your application with a web application firewall (WAF) that specifically analyzes web traffic and blocks any attempt to exploit vulnerabilities such as cross-site scripting or code injection.

* Scan for web-specific vulnerabilities, For example, security scanners, like Netsparker, ensure that no exploitable code gets to the production environment. But there could be logical vulnerabilities that can only be detected by manual code auditing.

* Turn off unnecessary functionality: A basic, common sense tip is to reduce the number of potentially vulnerable entry points. If attackers can exploit any of the components of the web server, the whole web server could be in danger.

* Keep an eye on server logs: Administrators should monitor them regularly to detect any suspicious behavior before it does any damage. By analyzing log files, you can uncover a lot of information to help you better protect the application.

* Use separate environments for development, testing, and production!

* Restrict access and privileges!
